# m0fdii: маленькая языковая модель с нуля

`m0fdii` — учебная генеративная языковая модель архитектуры decoder-only
Transformer. Она обучается предсказывать следующий UTF-8 байт и действительно
генерирует ответ токен за токеном. Это не словарь ответов и не система правил.

Проект намеренно не использует NumPy, PyTorch, TensorFlow, BLAS, pybind11 и
другие сторонние библиотеки. Тензоры, autograd, слои, оптимизаторы, checkpoint и
byte-level токенизатор реализованы здесь. C++20 backend подключается через
стабильный C ABI и стандартный `ctypes`.

## Быстрый старт на Linux

```bash
conda env create -f environment.yml
conda activate minillm
python tools/build_backend.py --release
python -m unittest discover -s tests -v
python tools/make_dataset.py
python train.py --config configs/debug.json
```

Интерактивный режим после обучения:

```bash
python chat.py --checkpoint checkpoints/debug.bin
```

Без собранной библиотеки автоматически используется медленный эталонный Python
backend. Явный выбор: `MINILLM_BACKEND=python` или `MINILLM_BACKEND=cpp`.
Число рабочих потоков C++ задаётся через `MINILLM_NUM_THREADS`.

## Что важно понимать

Первая модель имеет лишь сотни тысяч параметров, контекст 96 байт и обучается
на маленьком локальном корпусе. Она помогает изучить весь путь от float32-буфера
до генерации, но не является аналогом ChatGPT и не обладает общими знаниями.
Подробности устройства и эксплуатации дополняются вместе с реализацией.

## Почему модель может отвечать неправильно

- датасет очень мал;
- параметров на порядки меньше, чем у промышленных LLM;
- нет предварительного обучения на большом корпусе;
- контекст ограничен и измеряется байтами UTF-8;
- генерация вероятностна;
- фактическая точность ничем не гарантируется.
