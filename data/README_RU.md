# Данные

Проект обучает одну autoregressive модель на двух совместимых задачах:

1. `qa_seed.jsonl` → `train.txt`/`validation.txt`: продолжение вопроса ответом;
2. `text/train`/`text/validation`: продолжение обычного текста.

QA-файлы воспроизводимо создаются командой:

```bash
python tools/make_dataset.py --seed 42
```

Каждая JSON-строка содержит `question`, `answer` и необязательный массив
`variants`. Готовые примеры разделены пустой строкой и имеют маркеры
`Вопрос:`/`Ответ:`. Специальные BOS/EOS добавляет токенизатор при загрузке.

В `text/train` помещайте UTF-8 файлы `.txt`, `.md` или `.text`, на которых
разрешено обучать модель. В `text/validation` помещайте другие документы для
независимой оценки. Каталоги обходятся рекурсивно, каждый файл остаётся
отдельным документом. Примеры в репозитории малы и нужны для проверки механизма,
а не для обучения полноценному языку.

Доля обычных текстов задаётся полем `text_ratio` конфигурации или аргументом
`--text-ratio`. Несколько внешних корпусов можно подключить повторяющимися
`--text-train PATH` и `--text-validation PATH`.

Не добавляйте секретные, персональные или чужие защищённые данные. Проверяйте
лицензию, качество, кодировку и отсутствие дубликатов между train/validation.
